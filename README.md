# 小红书一体化爬虫

## 工具简介

小红书一体化爬虫是一个基于 Python 和 DrissionPage 开发的网页自动化采集工具，用于围绕指定关键词完成小红书搜索结果、帖子详情、评论内容和作者主页信息的分步采集。工具将原本分散的列表页爬取、详情页爬取和主页爬取逻辑整合为一个可连续运行的脚本，适合用于传播数据分析、内容特征研究、账号画像分析和社交媒体文本数据整理等场景。

脚本文件为：

```text
小红书一体化爬虫.py
```

## 核心功能

### 1. 关键词搜索结果采集

用户运行脚本后输入一个关键词，程序会自动打开小红书网页，等待用户完成登录，然后进入该关键词对应的搜索结果页。工具会模拟页面滚动，持续提取搜索结果中的帖子链接，并将链接保存为 CSV 文件。

采集字段包括：

- 关键词
- 帖子链接

### 2. 帖子详情信息采集

工具会根据搜索结果中提取到的帖子链接，逐条进入帖子详情页，采集帖子的核心内容信息。

采集字段包括：

- 关键词
- 详情链接
- 用户
- 用户主页
- 标题
- 正文
- 标签
- 发布时间
- 发布地点
- 点赞数
- 收藏数
- 评论数
- 图片链接

### 3. 评论信息采集

在爬取帖子详情时，工具支持继续采集评论区内容。用户可以在运行时设置每条帖子最多采集多少条评论，也可以输入 `0` 跳过评论采集。

采集字段包括：

- 帖子标题
- 评论用户
- 评论内容
- 评论时间
- 评论地点
- 评论点赞数
- 详情链接

### 4. 作者主页信息采集

帖子详情采集完成后，工具会从帖子数据中提取作者主页链接并去重。程序会询问用户是否继续爬取这些作者主页的详细信息，用户可以根据研究需要选择继续或跳过。

采集字段包括：

- 用户主页
- 昵称
- 小红书号
- 性别
- IP 属地
- 简介
- 标签
- 关注数
- 粉丝数
- 获赞与收藏数

## 运行流程

1. 启动脚本。
2. 按提示输入小红书搜索关键词。
3. 设置搜索结果页最多滚动次数。
4. 设置最多爬取多少条帖子详情。
5. 设置每条帖子最多爬取多少条评论。
6. 程序自动打开浏览器并进入小红书。
7. 用户在浏览器中手动完成登录。
8. 登录完成后回到终端按回车继续。
9. 程序自动进入关键词搜索结果页并滚动采集帖子链接。
10. 程序逐条打开帖子详情页，采集帖子内容和评论。
11. 帖子采集完成后，程序提示是否继续采集作者主页信息。
12. 程序将所有结果保存到本地 CSV 文件。

## 输出结果

每次运行时，工具会在脚本所在目录下自动创建一个带关键词和时间戳的结果文件夹，避免多次运行时覆盖历史数据。

输出目录格式示例：

```text
小红书_关键词_爬取结果_20260628_233000
```

目录内会根据实际采集情况生成以下 CSV 文件：

```text
小红书_关键词_搜索结果链接.csv
小红书_关键词_帖子详情.csv
小红书_关键词_帖子评论.csv
小红书_关键词_用户主页信息.csv
```

所有 CSV 文件均使用 `utf-8-sig` 编码，便于使用 Excel、WPS、Python 或其他数据分析工具打开。

## 环境要求

- Python 3.9 或更高版本
- Google Chrome 浏览器
- DrissionPage

安装依赖：

```bash
pip install DrissionPage
```

如果本机 Chrome 安装在默认路径：

```text
C:\Program Files\Google\Chrome\Application\chrome.exe
```

脚本会自动使用该路径。若 Chrome 安装路径不同，DrissionPage 会尝试使用默认浏览器配置，也可以在脚本中的 `DEFAULT_CHROME_PATH` 变量处自行修改。

## 使用方法

进入脚本所在目录：

```powershell
cd "你的项目目录"
```

运行脚本：

```powershell
python .\小红书一体化爬虫.py
```

随后根据终端提示输入关键词和采集参数，并在浏览器中完成登录。

## 预期效果

正常运行后，工具可以完成从关键词到帖子数据再到作者主页数据的完整采集链路，帮助用户快速获得结构化的小红书内容数据。采集结果可直接用于后续的数据清洗、文本分析、情感分析、话题聚类、互动量分析、账号特征分析和内容影响力研究。

预期产出包括：

- 一份关键词搜索结果帖子链接表
- 一份帖子详情结构化数据表
- 一份帖子评论结构化数据表
- 一份可选的作者主页信息表
- 一个按运行时间独立保存的数据结果目录

## 工具特点

- **一体化流程**：将搜索页、详情页、评论区和用户主页采集整合到一个脚本中。
- **交互式运行**：运行时由用户输入关键词、采集数量和评论数量，降低修改代码的需求。
- **登录友好**：保留手动登录步骤，适应需要账号状态的小红书网页访问场景。
- **分批写入**：帖子详情和评论数据按批次写入 CSV，降低长时间采集中断导致的数据损失。
- **结果隔离**：每次运行自动创建独立输出目录，避免覆盖已有数据。
- **依赖轻量**：除 Python 标准库外，仅依赖 DrissionPage，不强制依赖 pandas。

## 注意事项

本工具仅用于学习、课程研究和个人数据分析场景。使用时应遵守小红书平台规则、网站 robots 协议、相关法律法规和数据隐私要求。请合理控制采集频率和采集范围，不要用于批量骚扰、商业滥用、绕过访问限制或采集敏感个人信息。

由于小红书网页结构可能随时调整，页面选择器、搜索入口或字段位置发生变化时，脚本可能需要同步更新。若出现搜索失败、字段为空或评论无法加载等情况，建议先检查浏览器页面是否正常显示，再根据当前页面结构调整对应选择器。
